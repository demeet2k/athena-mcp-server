# Packet / Route / Burden / Replay Registry

## Purpose

This registry deepens the first 4D pass by defining the mature packet classes that actually move between folders and corpora.

## Route Families

### Root Route

`witness -> local artifact`

### Compression Route

`local artifact -> section -> crystal`

### Cortex Route

`crystal -> package -> overview`

### Runtime Route

`framework -> manifests -> active artifact`

### Mirror Route

`precursor build -> comparison -> absorption`

### Canopy Route

`crystal/package -> deeper network -> appendix/metro`

### Reseed Route

`meta-synthesis -> seed packet -> future intake`

## Burden Laws

1. A packet cannot jump two scales upward without a named support shell.
2. Mirror packets are useful evidence but do not outrank live witness packets.
3. Overview packets compress but may not erase frontier packets.
4. Runtime packets must carry burden honesty even when payload packets do not change.
5. Spore packets must preserve unresolved frontier explicitly rather than implying finality.

## Replay Classes

- `direct`: immediate witness or file replay
- `derivational`: recompute from operator law
- `topological`: replay through crystal line structure
- `package`: replay through ordered manuscript shell
- `runtime`: replay through manifest and queue state
- `reseed`: replay only by starting the next cycle

## Final Zero Point

The expanded registry says a mature corpus object is not just a file and not just a meaning. It is a packet family with a lawful route, an honest burden, and a specific replay class.
