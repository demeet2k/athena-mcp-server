# Voynich Skill Wishlist

## Purpose

This file is the preflight skill gap analysis for the full Voynich transcription and translation program.

It was produced after the dense folio framework, math kernel registry, and first `f1r` deep atlas were already in place.

## Framework State

Current strengths:

- the dense folio framework exists
- the canonical lens stack exists
- the VML operator registry exists
- the math kernel registry exists
- `f1r` already demonstrates the direct-ledger-plus-formal-overlay model

Current bottlenecks:

- raw EVA preparation is still too manual
- visual evidence does not yet have its own disciplined pass
- symbolic compilation is still partly artisanal
- honesty and completeness auditing are not yet formalized as a reusable skill
- corpus roll-up after each folio is still fragile and manual
- long-run autonomous continuation still needs a no-stop handoff layer
- blockers still risk being treated as hard stops instead of boundary-ledger problems
- finished folios still need stronger reciprocal packaging into the live master manuscript
- contradictory but still-admissible readings need a dedicated preservation skill

## Existing Skill Coverage

Already strong:

- `manuscript-intake` for corpus indexing
- `voynich-pharmacist` for five-book operational reading
- `domain-translator` for generic cross-domain render
- `manuscript-seed` for long-form synthesis
- `metro-cartography` for hidden transit lines
- `shadow-analysis` for blind spots
- `q-shrink` for compression

Not yet project-specific enough:

- raw EVA normalization
- visual codicology
- formal equation compilation across the canonical lens stack
- evidence-hygiene auditing
- post-folio corpus conduction

## Skill Wish List

### Priority 1

1. `voynich-meta-observer`
   Gap: no project-level skill that reviews framework, outcomes, and capability gaps before work begins.
   Why existing skills are not enough: current skills operate within domains, but none designs the active skill stack for the Voynich program itself.
   Action: create.

2. `voynich-eva-normalizer`
   Gap: no dedicated skill for extracting and preserving stable EVA line inventories.
   Why existing skills are not enough: transcription hygiene is more specific than generic intake or generic Voynich interpretation.
   Action: create.

3. `voynich-equation-compiler`
   Gap: no dedicated compiler from direct VML ledger to canonical math-lens equations.
   Why existing skills are not enough: generic domain translation does not enforce the actual local kernel registry or folio theorem structure.
   Action: create.

### Priority 2

4. `voynich-visual-codicologist`
   Gap: no disciplined visual grammar workflow for illustrated folios.
   Why existing skills are not enough: text-first skills do not structure plant, bath, zodiac, and rosette evidence rigorously enough.
   Action: create.

5. `voynich-confidence-auditor`
   Gap: no formal evidence-hygiene and completeness audit.
   Why existing skills are not enough: the project needs a release gate, not just interpretive critique.
   Action: create.

6. `voynich-corpus-conductor`
   Gap: no repeatable folio-to-corpus roll-up process.
   Why existing skills are not enough: existing synthesis skills do not manage the manuscript-order build state after each completed folio.
   Action: create.

### Priority 3

7. `voynich-autopoietic-continuator`
   Gap: no dedicated no-stop continuation skill that turns one completed folio into the next folio's intake.
   Why existing skills are not enough: execution planning exists, but it still needed a reusable runtime handoff law.
   Action: create.

8. `voynich-boundary-ledger`
   Gap: difficult folios and workflow stalls can still be mistaken for hard blockers when they may only be missing a hidden dependency or witness.
   Why existing skills are not enough: generic analysis skills do not explicitly classify boundary errors versus true blockers for this manuscript workflow.
   Action: create.

9. `voynich-reciprocal-rollup`
   Gap: completed folios still need a stronger guarantee that their outputs feed every higher-order artifact.
   Why existing skills are not enough: corpus conduction covers roll-up, but not the Chapter 11 reciprocity law across section, crystal, metro, and live master packaging together.
   Action: create.

10. `voynich-contradiction-preserver`
   Gap: conflicting but still admissible readings can collapse too early or stall progress.
   Why existing skills are not enough: the project needs a skill that preserves shared invariants while allowing honest forward motion under uncertainty.
   Action: create.

## Final Recommendation

Use this skill stack as the final pre-task layer before scaling beyond `f1r`:

1. meta observer to verify readiness
2. EVA normalizer to stabilize the next folio
3. visual codicologist if the folio is illustrated
4. equation compiler to build the symbolic layer
5. confidence auditor before release
6. corpus conductor after release
7. autopoietic continuator to trigger the next folio immediately
8. boundary ledger when a blocker appears
9. reciprocal rollup to feed the live master manuscript
10. contradiction preserver when evidence branches but work can still continue
