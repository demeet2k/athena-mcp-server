# Capsule 338 — Desire Compiler

**Source**: 2026-03-18_quantum_crystal_computing.md
**Family**: quantum_crystal_computing
**Lens**: F (Flower/Spectral)

The Desire Compiler D transforms a raw unknown U into the full execution package (Q_U, D_U, O_U, Π_U, Θ_U, G_U) via a six-stage pipeline:

1. U →[outline]→ C_U: Extract constraints from the unknown — what boundaries does U impose?
2. C_U →[attractor]→ D_U: Build the desire field from constraints — where does the solution want to live?
3. D_U →[address]→ Q_U: Assign a crystal address (QueryState) — locate the query in the lattice.
4. Q_U →[observer]→ O_U: Attach an observer to track convergence — who watches the collapse?
5. O_U →[quotient]→ Π_U: Compute the quotient projection — reduce symmetry to accelerate search.
6. Π_U →[commit gates]→ (Θ_U, G_U): Set commit tolerances and gates — when is the solution good enough?

The compiler is the bridge between raw intent (the unknown U) and the crystal's executable search. Each stage narrows the space: outline removes impossibilities, attractor builds the potential landscape, address localizes, observer instruments, quotient collapses symmetry, and commit gates set the stopping condition.

## Key Objects
- Desire Compiler D: U → (Q_U, D_U, O_U, Π_U, Θ_U, G_U)
- 6-stage pipeline: outline → attractor → address → observer → quotient → commit gates
- Constraint set C_U extracted from the unknown
- Observer O_U that tracks convergence
- Quotient projection Π_U that reduces symmetry
- Commit gates (Θ_U, G_U) that set tolerances

## Key Laws
- Every unknown must pass through all 6 compiler stages before execution
- The pipeline is strictly sequential — each stage depends on the prior
- The quotient step reduces symmetry; the crystal never searches symmetric copies
- Commit gates are set once at compile time, not adjusted during search
