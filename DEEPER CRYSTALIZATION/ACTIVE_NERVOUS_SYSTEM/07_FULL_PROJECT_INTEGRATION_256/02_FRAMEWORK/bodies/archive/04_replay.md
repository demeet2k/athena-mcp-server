# Archive Replay

## Role

ZIP-backed framework trees and historical payloads that remain hidden from ordinary browsing.

## Targets

- archive_atlas.json
- archive_manifest.json
- MATH framework archives

## 4 x 4 matrix

## File scale

- `archive.replay.file.seed`: Use `replay` on the single source artifact until it can define the object and create the first admissible representation.
- `archive.replay.file.link`: Use `replay` on the single source artifact until it can connect the object to parents, peers, and downstream surfaces.
- `archive.replay.file.prove`: Use `replay` on the single source artifact until it can add tests, receipts, witnesses, or quantitative closure.
- `archive.replay.file.publish`: Use `replay` on the single source artifact until it can promote the object into canonical operating surfaces.
