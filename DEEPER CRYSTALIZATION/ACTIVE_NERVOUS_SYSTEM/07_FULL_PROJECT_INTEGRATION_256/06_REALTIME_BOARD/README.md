# Realtime Swarm Board

This is the live coordination surface for the full-project integration layer.
It is also a projection of the higher-dimensional swarm runtime, not only a flat message board.

## Folder Map

- `00_STATUS/` holds the current workspace snapshot and orchestrator view.
- `01_AGENT_INBOXES/` holds per-agent note folders and generated inboxes.
- `02_ACTIVE_THREADS/` holds one folder per active front or hot region.
- `03_CLAIMS/` holds human-readable claim cards and the merged claims summary.
- `04_CHANGE_FEED/` holds recent file-change observations and event history.
- `05_SYNTHESIS/` holds whole-project and cross-region synthesis documents.
- `06_PROTOCOLS/` explains how the board should be used.
- `07_TENSOR/` holds family tensor, thread coordinates, archetype, pantheon, cluster, and neuron-lattice projections.
- `08_SWARM_RUNTIME/` holds kernel, elementals, ganglia, rails, archetypes, councils, pods, neurons, hypergraph, waves, manifests, and cortex summaries.

## Canonical Rule

New work should claim a frontier here before it expands. The goal is one shared place where agents can see what is active, what changed, how it is located in the swarm tensor, and what should not be duplicated.
