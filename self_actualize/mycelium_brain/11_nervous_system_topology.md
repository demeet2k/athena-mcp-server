## Routing Law

If the task is:

- a live manuscript section, route first through `tool_kit/02_section_final_draft_protocol.md`
- a whole-corpus deep pass, route first through `tool_kit/03_third_wave_corpus_synthesis_protocol.md`
- a tome build, route first through `tool_kit/04_tome_construction_contract.md`
- a dense recursive expansion, route first through `tool_kit/05_fractal_crystal_processing_mode.md`
- a live-memory request, route first through `tool_kit/01_live_docs_first_protocol.md`
- a tandem multi-agent build, route first through `tool_kit/07_tandem_agentic_loop_protocol.md`
- a ZIP-backed framework extraction, route first through `tool_kit/08_archive_promotion_protocol.md`
- a multi-agent shard merge, route first through `tool_kit/09_cross_agent_contraction_receipt_protocol.md`

## Folder Use Rule

No new prompt surface should be dropped loose into the workspace.
It should be normalized into `tool_kit/`, registered in `registry/`, and referenced from the metro index.
If multiple agents are active at once, the front should also be claimed in `registry/01_tandem_frontier_claims.md`.
If a frontier produces several shard artifacts, the merge should land with a contraction receipt before the front is marked done.

## Completion Condition

The nervous system is active when:

- prompts are stored as reusable files
- receipts record gate states and major refreshes
- maps point to the right files
- queues point to the highest-yield next action
- manifests carry a lawful restart seed
- family surfaces and rails make the swarm addressable
