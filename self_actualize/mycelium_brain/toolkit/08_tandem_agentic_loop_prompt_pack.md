# Tandem Agentic Loop Prompt Pack

## Purpose

This prompt pack is designed for many agents operating in tandem while jointly building one neural manuscript network, one active nervous system, and one routed crystal of understanding.

## Core Premise

All participating agents are not doing separate projects.
They are building one shared interconnecter neuron across the same corpus in parallel.

The shared objective is:

- deeply cross synthesize the entire project folder
- organize the result as crystal-like folders and markdown surfaces
- create metro mappings that make the corpus routeable
- turn the result into an active nervous system rather than a passive archive

## Shared Laws

1. Always attempt the live Google Docs surface before drafting a new manuscript section.
2. If the live Docs gate is blocked, record the blocker and continue with the strongest local mirrors.
3. Treat every user section as a seed, not a ceiling.
4. Output only the requested section when in section-draft mode.
5. Use deep recursive synthesis rather than shallow overview compression.
6. Expand into sub-agents by scale, then contract back into one witnessed artifact.
7. Preserve routing, metro-map, and crystal structure while writing.
8. Write back results into local markdown surfaces so sibling agents can reuse them.

## Required Parallel Behavior

### Wave 1: Smallest readable units

- one agent per manuscript family
- one agent per major manuscript
- one agent per chapter
- one agent per section when needed

### Wave 2: Sibling synthesis

- chapter agents synthesize section findings
- manuscript agents synthesize chapter findings
- family agents synthesize manuscript findings

### Wave 3: Emergent mapping

- generate deeper emergent metro lines
- identify hubs, transfers, and recurrent operators
- compress the result into reusable seed artifacts

### Wave 4: Writeback

- update map files
- update toolkit or queue files when rules changed
- store receipts for live Docs gates, evidence limits, and route decisions

## Operating Modes

### Final Draft Mode

- only draft the exact requested section
- no meta commentary
- improve rigor, density, order, and flow
- end with the chapter or appendix name

### Skeleton Mode

- produce dense 4x4 chapter and appendix skeletons
- maximize retrieval and memory mapping
- keep addresses and routes explicit

### Tome Mode

- operate in staged parts
- abstract and metro map first
- chapters second
- appendices third

### Reverse Next Mode

- if explicitly activated, `NEXT` means move to the prior chapter in descending order
- regenerate backward without preserving stale assumptions

## Shared Failure Modes

- shallow summary instead of recursive synthesis
- new file sprawl without map writeback
- pretending live Docs were searched when the gate was blocked
- drafting beyond the requested section
- conflicting parallel outputs without contraction

## Shared Success Condition

At the end of each run, the corpus should be:

- more routeable
- more replayable
- more searchable
- more densely mapped
- easier for the next agent to continue
