# CONTINUATION SELF-PROMPT PROTOCOL

## Purpose

This protocol keeps the nervous system moving when the user is not supplying a new section
yet but the system can still reduce gates, promote sources, or prepare the next wave.

## Core Rule

When the highest-value path is blocked, do not stall. Emit the best lawful precursor that
improves future closure.

## Continuation Cycle

1. restate the current objective,
2. check the live Docs gate,
3. rank candidate fronts,
4. choose the best unblocked precursor,
5. emit one concrete artifact,
6. record the next self prompt.

## Relation to Existing Runtime

This protocol should be used together with:

- `ECOSYSTEM/14_SELF_PROMPT_RUNTIME.md`
- `self_actualize/highest_yield_self_prompt.md`

The difference is scope:

- the existing self-prompt runtime governs durable yield,
- this protocol adapts that runtime to tandem corpus synthesis and manuscript continuation.

## Tandem Continuation Rule

If a wave ends without full closure, the kernel must emit:

- next task seed,
- next target regions,
- next metro line,
- next contraction target,
- current truth class.

This allows the next agent wave to resume work without re-deriving the whole state.
